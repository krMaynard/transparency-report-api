方法说明

本数据集如何采集、处理并实现安全查询——以及它的局限所在。

这是什么

这是一个对公开透明度报告数据的独立、非官方聚合，不隶属于任何平台、公司或政府。它将两类报告整合到一个可查询的模式中：

  • 欧盟 DSA 透明度报告
    在线平台依据欧盟《数字服务法》（DSA）公布的内容审核统计数据，遵循 DSA 实施条例模板的表 3–11。涵盖被指定的超大型在线平台和搜索引擎（VLOPs / VLOSEs），以及数十个使用统一模板报送的较小平台。在 DSA 报告仪表板查看实时计数和覆盖时段。
  • Google 政府删除请求
    Google 报告的政府删除内容请求数量，按国家/地区、产品、原因和报告时段统计。在政府删除请求仪表板中探索。

每个平台在何处发布其原始报告，已编入报告位置页面，并附有源文件存档副本的链接。

数据来自何处

每一项数据都源自发布它的平台或公司——这些是它们报告的数字，在此原样转载，而非本项目产生的估算。本项目的贡献在于聚合与结构化：收集各自发布的报告，提取其表格，并将它们规范化到一个模式中，以便比较和查询。没有任何内容经过编辑性调整；唯一的转换就是下文所述的机械化提取和规范化步骤。

一份报告如何变得可查询

每份源报告的表格被提取并载入一个只读的 SQLite 星型模式——每个 DSA 报告表（t3–t11）对应一个事实表，周围环绕共享的维度表（服务、类别、章节、指标、范围、界面、报告时段）。重复的标签被内化为小整数 id，因此同一类别或平台名称只存储一次，并在各处引用。在模式页面浏览生成的表格和所有可查询字段。

避免重复计数

DSA 模板会把一个汇总总计行紧挨着它所汇总的分项行放置（例如，在按类别计数旁边放一个总计的“All the entries”类别，或在维持/撤销结果旁边放一个“Total number”范围）。对这样的列进行简单的 SUM 会把各分项和它们自身的总计一起计数，从而虚增数字。加载器为每个汇总行打上 is_total 标记，并将其作为可筛选的维度公开，因此查询可以有意选定单一粒度——只取总计，或只取分项——绝不会把总计与其组成部分相加。仪表板的标题数字使用报告的总计；其分项图表使用组成行。若某项数字仍带有注意事项，API 会返回一条 warnings 说明，并由仪表板在结果旁一并呈现。

跨语言阅读

非 VLOP 平台以任何一种欧盟官方语言报送统一模板，因此同一个模板行会以不同的文本到达（Decisions upheld / Décisions confirmées / Bestätigte Entscheidungen）。每一行都保留其原始语言标签用于显示，但同时被标注一个语言中立的规范键，该键学习自一个对照表——它将相同结构的报告对齐到一份英文参照，并剔除任何含糊不清的内容。查询可以按规范键分组或筛选，从而跨语言进行同类比较，同时仍显示原始文本。

如何查询

这里没有 SQL 接口。查询是一个结构化请求——它指定一个表、一个由字段/操作/值条件组成的布尔筛选器，以及可选的分组、聚合、排序和限制。服务器会对照该表固定的注册表校验每个字段和操作，并将其编译为单条参数化的 SELECT；用户值始终以绑定方式传入，绝不拼接。跨表“复合”查询通过同一校验边界，在共享维度上连接两个或更多表。即便是自然语言“提问”框，也只是提议一个结构化查询——随后它会像其他任何查询一样先经过校验再运行。在 DSA 报告仪表板上试用，或阅读 API 参考。

可复现性与引用

该数据库是一个静态、只读的快照。每个快照都带有一个不可变的数据集版本标记（同时通过 ETag 和 X-Dataset-Version 标头发送），并且每个事实行都公开其源报送的 report_id。同时引用两者——数据集版本 + report_id——可以精确锁定某项数字背后的具体行，以便重新获取。查询结果在排序或分页时以确定的顺序输出，因此保存的提取结果可在字节层面复现。快照的构建日期和覆盖时段显示在每个仪表板的“引用为”行中，以及每次导出的标头里。

已知局限

  • 快照，而非实时
    数字反映构建快照时可获得的报告；后续的更正或新报送的报告只会出现在随后的快照中（并附带新的数据集版本）。
  • 源质量参差不齐
    各平台报送的严谨程度和格式各不相同。被错误解析的无效行（表头单元格、空白、某些提取泄漏的杂散值）在加载时会被剔除，而不是凭猜测处理。
  • 部分标签未规范化
    希腊语提取在源数据中存在列位移，因此大多数希腊语指标/范围标签被保留为正确但未映射的状态，以免冒错误映射的风险；类别标签尚未跨语言对照。
  • 演示服务
    这是一个作品集/演示项目。它忠实于其来源，但并非官方记录——对于任何重要事项，请始终对照平台自己发布的报告（可从报告位置链接进入）加以确认。
