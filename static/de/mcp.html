Integrationen

MCP-Server

Verbinden Sie Claude Desktop oder Claude Code direkt mit dem DSA-VLOP-Transparenzdatensatz.

Der native MCP-stdio-Server (mcp_server.py) stellt 8 Werkzeuge bereit, damit ein KI-Agent Transparenzdaten erkunden, abfragen und exportieren kann, ohne SQL zu schreiben oder Dokumentation zu lesen — beginnen Sie mit list_tables, erstellen Sie eine Abfrage und erhalten Sie die Ergebnisse in einem Schritt.

Wie es funktioniert

Der MCP-Server ist ein leichter HTTP-Client über der laufenden API: Jeder Werkzeugaufruf entspricht einem echten Endpunkt, sodass alle Abfragen dieselbe compile_query-Validierung durchlaufen wie die Web-UI und die REST-API — SQL wird nie akzeptiert, jedes Feld und jede Operation wird gegen das Tabellen-Registry geprüft. Der Server hat einen minimalen Abhängigkeits-Fußabdruck (mcp + httpx) und importiert niemals die FastAPI-App.

Claude Desktop / Claude Code
        ↕ stdio (MCP)
   mcp_server.py
        ↕ HTTP (httpx)
   Transparency Report API  →  SQLite DB

Schnelleinrichtung

1
In einer separaten virtuellen Umgebung installieren 
python -m venv .venv-mcp
source .venv-mcp/bin/activate   # Windows: .venv-mcp\Scripts\activate
pip install mcp httpx
2
API starten (oder auf die gehostete Instanz verweisen) 
uvicorn main:app --port 8000
3
API-Schlüssel erhalten (benötigt für submit_query, poll_job, ask)

Rufen Sie register aus Ihrem MCP-Client auf oder verwenden Sie das Forschungsportal — für Demo-Schlüssel ist keine Anmeldung erforderlich.

Werkzeuge

WerkzeugAuthentifizierungBeschreibung
list_tables public Listet alle abfragbaren DSA-Berichtstabellen mit Dimensionen, Kennzahlen und verfügbaren Aggregatfunktionen auf. Hier beginnen.
describe_table public Beschreibt die Felder einer Tabelle, gültige Operationen und eine ausführbare Beispielabfrage. Gibt das vollständige Feld-Registry zurück, wenn ein API-Schlüssel gesetzt ist.
dataset_overview public Zusammenfassende Kennzahlen: Berichtszeitraum, Plattformanzahl, Gesamtzahl der Artikel-16-Meldungen, Top-Plattformen und -Kategorien.
run_query public Führt eine strukturierte Abfrage synchron über POST /api/explore aus. Zeilenlimit; kein API-Schlüssel erforderlich. Gut für schnelle Erkundungen.
ask API key Stellt eine Frage in natürlicher Sprache. Ein LLM übersetzt sie in eine strukturierte Abfrage (nie SQL), die dieselbe Validierung wie run_query durchläuft.
register public Für einen Demo-API-Schlüssel registrieren (ruft POST /api/portal/register auf). Gibt {"api_key": "rk_…", "expires_at": "…"} zurück. Keine Anmeldung erforderlich.
submit_query API key Sendet eine strukturierte Abfrage an die asynchrone Job-Warteschlange (POST /api/query). Kein Zeilenlimit. Gibt sofort eine job_id zurück.
poll_job API key Fragt einen eingereichten Job ab, bis er abgeschlossen ist, und gibt dann die Ergebniszeilen zurück. Verwendet exponentielles Backoff; konfigurierbarer Timeout (Standard 60 s).

Konfiguration

Claude Desktop

{
  "mcpServers": {
    "transparency": {
      "command": "/path/to/.venv-mcp/bin/python",
      "args": ["/path/to/mcp_server.py"],
      "env": {
        "TRANSPARENCY_API_URL": "http://127.0.0.1:8000",
        "TRANSPARENCY_API_KEY": "your-key"
      }
    }
  }
}

Claude Code

claude mcp add transparency \
  /path/to/.venv-mcp/bin/python \
  /path/to/mcp_server.py \
  -e TRANSPARENCY_API_URL=http://127.0.0.1:8000 \
  -e TRANSPARENCY_API_KEY=your-key
VariableDefaultDescription
TRANSPARENCY_API_URL http://127.0.0.1:8000 Basis-URL der laufenden API.
TRANSPARENCY_API_KEY — Ihr API-Schlüssel. Erforderlich für submit_query, poll_job, ask und das vollständige describe_table-Feld-Registry.
TRANSPARENCY_API_TIMEOUT 30 Timeout pro Anfrage in Sekunden.
